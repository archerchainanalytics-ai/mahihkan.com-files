On the Record — What This Is

INxS™ is not a strategy generator. It is not a chatbot that suggests trades. It is not machine learning deciding what to do with your money. It is a deterministic, modular automated trading bot factory built on a locked 7-module async pipeline. Every module has a defined input, a defined output, and a defined behavior. The license validates locally via HMAC-SHA256 — no network calls, no cloud dependency, no third-party validation that can go down at 3am when a position is open. It is actual software that does exactly what the architecture says it does, every time.

Origin

Why This Exists

Trading bot infrastructure is either locked inside a platform you don't control, or it's an open-source project with no commercial structure, no licensing model, and no path to white-label deployment. Neither option works for operators who need to move fast, brand independently, and distribute to clients without rebuilding from scratch every time.

INxS™ was built to be the factory, not the bot. You configure it once per client. You apply branding. You generate a license key. You zip and deliver. The client gets a fully operational, independently licensed trading system running under their brand. You get a repeatable delivery pipeline that scales without proportional effort.

Architecture

7-Module Locked Pipeline

INxS™ operates on a fixed 7-category architecture. Every bot assembled through the factory contains exactly one module from each category, wired together via an async pipeline orchestrated by the Assembler. Categories cannot be removed or reordered. Modules within each category are swappable.
